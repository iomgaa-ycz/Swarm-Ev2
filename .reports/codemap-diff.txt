## 主要变更

### 1. 基因系统重设计 (commit 02a3294)
- 旧: 7 基因 (DATA, MODEL, LOSS, OPTIMIZER, REGULARIZATION, INITIALIZATION, TRAINING_TRICKS)
- 新: 4 基因 (DATA, MODEL, TRAIN, POSTPROCESS) + GENE_SUB_ASPECTS 精细化变异
- 每个基因有 3-4 个 sub-aspect（共 15 个可变异维度）

### 2. 两阶段进化架构 (commit 6c80bc6)
- Phase 1 (Draft): 生成初始种群，直到达到 phase1_target_nodes
- Phase 2 (GA): 50% merge + 50% mutate，遗传算法优化
- Orchestrator 新增 run_epoch_draft() / run_epoch() 双入口
- Node 新增字段: dead, debug_attempts, approach_tag, generation

### 3. Review 系统重写 (commit 5ac4f9b)
- 旧: Function Calling (GLM-4.6 submit_review tool)
- 新: 纯文本 JSON 解析 + 前置 LLM 输出压缩
- text_utils.py 大幅扩展 (72→143 行)

### 4. 新增 gene_compatibility.py (141 行)
- 框架互斥检测 (torch vs sklearn vs tensorflow)
- CompatibilityResult dataclass
- merge 前注入兼容性警告

### 5. 任务类型变更
- 旧: explore, merge, mutate
- 新: draft, merge, mutate (explore 重命名为 draft)
- Prompt 模板: explore.j2 → draft.j2

### 6. P0 Bug3 修复 (commit d22d7a8)
- neg_RMSE 负号问题修复
- 新增 _sanitize_metric_value() 入口校验

### 7. gene_selector.py 大幅扩展 (322→448 行)
- 退化检测 (pheromone_with_degenerate_check)
- 主父代推断 (get_primary_parent)
- 非 stub 基因选择

### 8. stdout 截断策略改进
- 旧: 简单截断
- 新: 保留头 N + 尾 M 字符（避免丢失关键输出如 "Validation metric:" 行）
