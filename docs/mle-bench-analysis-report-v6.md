## 3. 发现的三个系统级 Bug

### Bug 1: Prompt 截断导致 LLM 误读 metric（影响: denoising 丢银牌）

**影响竞赛**: denoising-dirty-documents（V5 银牌→V6 无牌）

**根因链条**:
1. 节点 f799c849 的训练输出过长（322,233 字符），5 折 × 60 epoch 的进度条
2. Prompt 截断后，`Validation metric: 0.129663` 被截掉
3. LLM 只看到历史最佳 `"Current Best: 0.0211"`，误以为是本节点 metric
4. 系统以 metric=0.021018 选中该节点作为最终提交
5. 该节点真实 RMSE=0.129663，模型退化为"全白预测"（输出均值=0.994）

**真正的最佳节点**: 473f1342（step=8），5 折 RMSE=[0.021534, 0.019951, 0.021762, 0.020433, 0.021605]，若提交预计得分 ~0.021（银牌水平）。

**修复方案**: stdout 截断策略改为**保留头尾**（头部 N 字符 + 尾部 M 字符），确保 `Validation metric:` 行始终出现在 Prompt 中。

### Bug 2: `_parse_metric_from_stdout` 匹配 "Std Validation metric"（影响: spooky-author 虚假好成绩）

**影响竞赛**: spooky-author-identification

**根因**:
代码输出两行:
```
Validation metric: 6.018671      ← 真正的 CV log_loss 均值
Std Validation metric: 0.211086  ← 标准差
```

正则 `r"Validation metric:\s*(...)"`  匹配了两行（"Std Validation metric" 包含子串 "Validation metric"），取 `matches[-1]` = 0.211086（std），导致系统记录 metric=0.211 而非真实的 6.019。

**后果**: 系统认为 spooky-author 有 0.211 的好成绩，实际 LB=5.605（差铜牌线 5.31）。

**修复方案**:
```python
# 改为行首匹配
r"^Validation metric:\s*([-+]?\d*\.?\d+(?:[eE][-+]?\d+)?)"  # + re.MULTILINE
```

### Bug 3: sklearn `neg_` scoring 负号未处理（影响: nomad2018 选错最佳节点）

**影响竞赛**: nomad2018-predict-transparent-conductors（差铜牌仅 0.00007）

**根因**:
节点使用 `cross_val_score(scoring="neg_root_mean_squared_error")` 返回负值 RMSE：
```
Validation metric: -0.063126   # sklearn 约定: neg 前缀表示取负
```

Review Agent 行为不一致:
- Draft 节点: LLM 理解了取绝对值 → 存储 0.0631（正确）
- Mutate 节点: LLM 直接存储 -0.063126（错误）

系统在 `lower_is_better=True` 模式下选 metric 最小值 → 选中了 -0.065850 的节点（负数最小），但该节点真实 RMSLE=0.065850，并非最佳。

**真正的最佳节点**: 11ff595c（draft，真实 RMSLE=0.051115），该值远好于铜牌线 0.06582，**可冲银甚至金牌**。

**修复方案**: 在 metric 存储时，检测负值并取绝对值（RMSLE 不可能为负）。或在 Prompt 中要求 `Validation metric` 必须打印正值。

### 4.2 逐竞赛分析

#### nomad2018（差距 0.00007，**Bug 导致丢牌**）

- **现状**: 提交了 d649a401（RMSLE=0.06585），真正最佳是 11ff595c（RMSLE=0.05112）
- **GA表现**: 35 merge 全部聚集在 ~0.076（退化），mutate 最佳 0.06060 < draft最佳 0.05112
- **SOTA对比**: 金牌用晶体图n-gram+核岭回归（需领域专业知识），我们的通用GBDT已接近无领域知识上限
- **修复收益**: 修Bug后提交正确节点 → **大概率银牌级**（0.051 远好于铜 0.066）

#### jigsaw-toxic（差距 0.001，正常CV-LB gap）

- **现状**: 正确提交了最佳节点 8c613b96（内部 0.990，LB 0.985），gap=0.005 属合理范围
- **GA表现**: merge 全退化（0.966~0.968），mutate 最佳 0.976 < draft 最佳 0.990
- **SOTA对比**: 2018年获奖方案用 Bi-GRU + 多词向量 + **回译数据增强**。我们用 DistilBERT(MAX_LEN=128)，理论更强但 token 截断和缺乏数据增强制约了性能
- **改进方向**: MAX_LEN 256→512 + 回译增强 + 伪标签

#### tabular-may（差距 0.007，特征工程瓶颈）

- **节点数仅 10**（40% buggy），3个TimeoutError，GA未触发
- **SOTA对比**: 冠军发现 f_27 是10字符字符串，可拆解为字符级交互特征 + 三分区间特征
- **我们的差距**: 自动化ML无法发现字符串列的字符级语义，这是领域发现型瓶颈
- **改进方向**: 有限。除非加入字符串特征探索策略

#### leaf-classification（差距 0.008，小数据问题）

- **81个节点**（GA有53个merge+mutate），但 merge buggy率 53.1%
- **SOTA对比**: 冠军用 LR + 混淆类专项 RF，极简方案胜过复杂集成
- **我们的差距**: 99类×16样本=极小数据集，复杂模型严重过拟合，应用简单LR
- **改进方向**: Prompt 中加入"小数据集优先简单模型"的策略引导

#### aptos（差距 0.013，**从零提交到有分数的重大改善**）

- **V5零提交→V6有分数**（0.902），自适应超时部分生效
- **SOTA对比**: 金牌用 EfficientNet + Ben预处理(CLAHE对比度增强) + 2015历史数据预训练 + MSE回归(非分类)
- **我们缺失**: Ben预处理、历史数据、回归建模（我们可能用了分类）
- **改进方向**: 图像竞赛模板中加入 Ben预处理、回归建模等标准trick

#### random-pizza（差距 0.031，**数据泄漏致过拟合**）

- **内部 0.797 远超铜 0.692**，但 LB 仅 0.661
- **根因**: `giver_username_if_known` 字段在训练集中泄漏标签（有值=已收到披萨），但测试集全为 "N/A"
- 模型学会了依赖该字段，测试时该字段无信息 → 所有预测输出 ~0.765（几乎无区分度）
- **修复方向**: Prompt 中加入"排除测试时不可用特征"的警告

### 5.2 GA 产出质量分析（核心问题：GA 几乎不产生更好结果）

| 竞赛 | Draft最佳 | Merge最佳 | Mutate最佳 | GA超越Draft? |
|------|----------|----------|-----------|-------------|

| random-pizza | **0.797** | 0.720 | 0.727 | ❌ 全退化 |
| spooky-author | 0.337 | 0.435 | 0.333 | ≈ 几乎持平 |

**统计**: 14个GA活跃竞赛中:
- GA 超越 Draft: **2-3个**（detecting-insults的mutate、whale/mlsp的merge）
- GA 退化: **8-9个**
- GA 持平: **2-3个**

**结论**: **merge 操作整体上严重退化**，尤其在 nomad/jigsaw/denoising 中引入了比 draft 差得多的结果。mutate 偶有改善（detecting-insults），但总体也不如 draft。

### 5.3 GA 退化原因分析

1. **Merge 策略盲目混合**: 将两个方案的基因段拼接，但不同方案的数据处理/模型/训练逻辑不兼容，拼接后代码不协调
2. **Mutate 过于保守**: 随机选一个基因段变异，但变异方向随机，无定向优化信号
3. **过拟合放大**: GA 选择 "内部 metric 最高" 的个体繁殖，但内部 metric 虚高（数据泄漏、metric 误读）会被选择压力放大
4. **缺乏泛化约束**: GA 没有 CV-LB gap 检测，无法识别过拟合个体

## 7. V7 改进建议（按优先级排序）

### P0: 修复三个系统级 Bug（预期 +3~4 枚奖牌）

| Bug | 修复 | 预期影响 |
|-----|------|---------|
| Prompt 截断策略 | stdout 截断保留头尾（头N+尾M字符） | denoising 恢复银牌 |
| Std Validation metric 匹配 | 正则改为行首匹配 `^Validation metric:` | spooky-author 正确报告 |
| neg_RMSE 负号 | metric 存储时取绝对值 | nomad 恢复银/金牌 |

**预计修 Bug 后**: denoising 🥈 + nomad 🥈/🥇 + (spooky 未必得奖但不再误判) = **+2 枚**

### P1: GA 策略大幅改进（预期 +2~3 枚奖牌）

当前 GA 的核心问题是 **merge 退化 + 选择压力放大过拟合**。

| 改进 | 方案 | 预期 |
|------|------|------|
| **merge 兼容性检查** | merge 前检查两个 parent 的框架兼容性（同为 sklearn/DL/GBDT），拒绝跨框架 merge | 减少 merge 退化 |
| **mutate 定向化** | 基于 error analysis 引导 mutate 方向（而非随机） | 提升 mutate 成功率 |
| **过拟合检测** | 增加 train-val gap 检测，gap > 阈值的节点不参与繁殖 | 减少过拟合放大 |
| **精英保留改进** | Draft-only 最佳节点始终保留为提交候选，即使 GA 节点 metric 更高 | 避免 GA 选错覆盖好结果 |

### P2: Prompt 与策略优化（预期 +2~4 枚奖牌）

| 改进 | 目标竞赛 | 方案 |
|------|---------|------|
| **数据泄漏检测** | random-pizza 类 | Prompt 中加入"排除测试时不可用字段"警告 |
| **小数据策略** | leaf-class 类 | 检测样本数<2000时，优先LR/RF，禁用复杂DL |
| **图像竞赛模板** | aptos/siim/ranzcr | 加入 Ben预处理、回归建模、预训练模型选择等标准trick |
| **文本竞赛模板** | jigsaw/text-en/text-ru | 回译增强、MAX_LEN≥256、伪标签策略 |
| **超时预防** | 全局 | 按模型复杂度而非数据大小调整超时；DL模型先小epoch试跑 |

### P3: 架构级改进（长期）

| 改进 | 方案 |
|------|------|
| **多提交策略** | 保存 Draft 最佳 + GA 最佳两个提交，选 CV 更稳定的那个 |
| **Ensemble** | 对 Top-3 节点做加权平均预测 |
| **CV 稳定性度量** | 使用 fold std 作为过拟合风险指标 |

## 9. 总结

### V6 的成功
1. **GA 触发率**: 4.5% → 63.6%（4基因+K=4 生效）
2. **新增2枚奖牌**: mlsp-birds(🥈), whale(🥈)
3. **aptos 从零提交到有分数**: 自适应超时部分起效

### V6 的失败
1. **三个系统 Bug** 导致 denoising 丢银牌、nomad 丢金牌、spooky 误报metric
2. **GA 产出质量极差**: 14个GA竞赛中，仅2-3个真正超越 Draft
3. **TimeoutError 恶化**: 占比从 29.7% → 44.4%
4. **detecting-insults 严重过拟合**: GA 选中的高 metric 节点在 LB 上退化（0.947→0.794）

### 达成 80% 获奖率的路径

| 阶段 | 改进 | 预期增量 | 累计 |
|------|------|---------|------|
| 当前 | — | 8/22 = 36.4% | 36.4% |
| P0 (修Bug) | +denoising🥈 +nomad🥈 | +2 | 10/22 = 45.5% |
| P1 (GA优化) | +detecting-insults恢复🥇 +jigsaw🥉 | +1~2 | 12/22 = 54.5% |
| P2 (Prompt策略) | +random-pizza +leaf +aptos | +2~3 | 14-15/22 = 64-68% |
| P3 (架构+Ensemble) | +text-en +tabular-may +其他 | +2~3 | 17-18/22 = 77-82% |

**关键认知**: 当前最大瓶颈不是算法能力，而是**系统工程质量**（Bug、metric解析、提交选择）。修复 P0 三个 Bug 即可立刻获得 +2 枚奖牌，投入产出比最高。
